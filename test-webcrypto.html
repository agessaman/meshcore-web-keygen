WebCrypto Ed25519 Performance Test

Test Strategy
This test compares FOUR approaches for Ed25519 vanity key generation: 1. WebGPU Implementation (GPU-accelerated) - Custom WGSL shaders - ~1,300 keys/sec (measured - poor performance) 2. JavaScript: noble-ed25519 (CPU, single-threaded) - Pure JavaScript - Measured: ~26,000 keys/sec 3. JavaScript: noble-ed25519 + Web Workers (CPU, multi-threaded) - JavaScript on multiple CPU cores - Measured: ~37,000 keys/sec (1.4x speedup) 4. WebAssembly: libsodium.js (CPU, single-threaded) - WASM-compiled C code - Expected: ~60,000-80,000 keys/sec (2-3x faster than JS) 5. WebAssembly: libsodium.js + Web Workers (CPU, multi-threaded) - WASM on multiple CPU cores - Expected: ~600,000-800,000 keys/sec (10 cores × 60-80K/sec) Target prefix: "CA" (2 hex characters) Batch size: 250,000 keys per test CPU cores: ${navigator.hardwareConcurrency || 'unknown'}
